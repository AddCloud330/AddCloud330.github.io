Thinking

Thinking 0.1


  • 新建时，处于为未跟踪状态（untracked）
  • 添加内容并用add命令之后，文件处于暂存状态（staged）
  • 在修改之后，处于被修改状态（modified）

修改后和add前的status不同，前者是未跟踪状态，不在git管理范围内；后者已经被跟踪，修改之后处于被修改状态，只是修改还没有提交。

Thinking 0.2

  • add the file : git add
  • stage the file : git add
  • commit : git commit

Thinking 0.3

  • 被错误删除： git restore print.c
  • 被错误删除且git rm print.c： 先 git restore –staged print.c 再 git restore print.c
  • 将hello.c移出暂存区： git restore –staged hello.c

难点分析

Exercise 0.1

需要掌握判断回文数的c语言实现，以及Makefile、shell脚本、文件操作的基本用法。

Exercise 0.2

用到了Shell脚本的循环和条件判断语法，批量处理目录，需要避免循环或通配符使用错误。

Exercise 0.3

使用grep和awk命令，将它们结合起来实现精准的识别和文本提取。

Exercise 0.4

用sed命令实现文件中指定字符串的全局替换，兼容任意合法文件名和替换字符串；涉及需配置跨目录编译规则多级Makefile。

实验体会

lab0主要涉及CLI的操作，我一开始觉得非常不习惯，很不直观。但是在完成实验的过程中，学习了文本操作、脚本、Makefile等知识后，我发现使用CLI解决一些批量的、需要泛化的工作时，实际上是效率大大提高的。但是bash毕竟是一门新的语言，还有很多细节等待我们探索，我认为我还需要更多的实践去不断熟悉CLI的操作。
